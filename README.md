# Course-Work
"РАЗРАБОТКА АВТОМАТИЗИРОВАННОЙ СИСТЕМЫ ДЛЯ ОРГАНИЗАЦИИ РАБОТЫ ПО КУПЛЕ - ПРОДАЖЕ МЕДИАФАЙЛОВ"

Общие требования. Целью курсового проекта является разработка объектной модели и ее программная реализация на языке C++. При разработке объектной модели требуется определить ее составные части, установить и спроектировать корректные связи между ними. В результате выполнения проекта должна быть оформлена пояснительная записка в соответствии со Стандартом предприятия БГУИР (https://www.bsuir.by/m/12_100229_1_185586.pdf).  
Разработанный курсовой проект должен быть представлен в виде консольного приложения. В рамках работы над курсовым проектом обязательно должны быть реализованы следующие задачи: 
1)	Разработка пользовательского меню, включающего следующие опции: 
­ разделение ролей, авторизация пользователей, хранение пароля в зашифрованном виде;  
­ просмотр, добавление, редактирование и удаление записей; 
­ поиск, сортировка и фильтрация записей; 
­ реализация заказа (вывод данных в виде отчета); 
­ другие опции в зависимости от специфики автоматизируемой задачи. 
2)	Хранение исходной и итоговой информации в текстовых или/и бинарных файлах. Для хранения данных в оперативной памяти использовать контейнеры библиотеки С++ STL 
(Standard Template Library).  
3)	Использование в программе следующих программных конструкций:

­   реализация базовых принципов объектно-ориентированного программирования (инкапсуляции, наследования, полиморфизма); 
­   использование абстрактных классов; 
­   использование передачи параметров по ссылке и по значению; 
­   использование встроенных, пользовательских, дружественных, виртуальных функции;  
­   использование пространств имен (встроенных и собственных); 
­   реализация обработки ошибок программы (средствами языка С++); 
­   использование перегрузки методов и операторов, переопределение методов; ­ использование шаблонов классов и методов; ­ использование статических методов и полей. 
­   использование динамического выделения памяти в пользовательских классах и умных указателей (smart pointers) языка С++. 
­   использование потоков С++, перегрузки операторов ввода/вывода для пользовательских классов, контроля ошибок при работе с потоками ввода/вывода. 

Требования к программному обеспечению: 
­ операционная система Windows 10 и выше, macOS 12 и выше; 
­ интерфейс программы и данные должны быть представлены на русском или белорусском языках; 
­ приложение должно запускаться без использования интегрированных средств разработки. 
При проектировании программного обеспечения необходимо выполнить моделирование с использованием следующих стандартов: 
­ IDEF0 для функционального моделирования процессов предметной области (контекстная диаграмма + не менее 3-х уровней декомпозиции контекстной диаграммы); 
­ UML 2.0 и выше для создания моделей представления системы (диаграмма классов, диаграмма вариантов использования); 
­ «Стандарт предприятия. Дипломные проекты (работы). Общие требования СТП 01– 2024», г. Минск, БГУИР, 2024. 
4. Содержание пояснительной записки (перечень вопросов, которые подлежат разработке):

3.	Функциональное моделирование на основе стандарта IDEF0. 
4.	Структура используемых данных (описание в каком виде хранятся данные, как данные обрабатываются); 
5.	Описание созданных программных конструкций (код всех конструкций из п.3.3 задания и его описание); 
6.	Разработка и описание диаграммы классов приложения; 
7.	Разработка и описание диаграммы вариантов использования приложения (для всех актеров приложения); 
8.	Блок-схема алгоритма работы всей программы и двух основных методов (3 алгоритма); 
9.	Описание алгоритма запуска приложения, его использования, результаты работы программы, тестирования обработки ошибок в приложении; 
10.	Заключение.  
11.	Список использованных источников.  
12.	Приложения (листинг основных блоков программы с комментариями к ним, диаграмма классов, диаграмма вариантов использования, блок-схемы алгоритмов).
